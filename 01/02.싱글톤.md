## 🏗️ 싱글톤 패턴 (Singleton Pattern)

**싱글톤 패턴**이란 애플리케이션 실행 시 특정 클래스의 인스턴스가 **오직 하나만 생성**되도록 보장하고, 어디서든 그 인스턴스에 접근할 수 있도록 하는 디자인 패턴입니다.

---

### 1. 핵심 구현 원칙
1.  **private 생성자**: 외부에서 `new` 키워드로 객체를 생성할 수 없도록 제한
2.  **static 변수**: 클래스 레벨에서 단 하나의 인스턴스를 유지
3.  **static 메서드**: 외부에서 인스턴스에 접근할 수 있는 유일한 통로를 제공

---

### 2. 자바 구현 코드 (Eager Initialization)

```java
/**
 * 클래스 로딩 시점에 인스턴스를 바로 생성하는 방식
 */
public class Singleton {

    // 1. static 영역에 객체를 미리 생성 (private으로 설정하여 외부 접근 차단)
    // final 키워드를 붙여 인스턴스 교체를 방지
    private static final Singleton instance = new Singleton();

    // 2. 생성자를 private으로 선언하여 외부에서 new 사용 방지
    private Singleton() {
        // 인스턴스 초기화 로직
    }

    // 3. 외부에서 인스턴스를 요청할 때 사용할 public static 메서드
    public static Singleton getInstance() {
        return instance;
    }

    // 예시용 비즈니스 로직 메서드
    public void doSomething() {
        System.out.println("싱글톤 인스턴스 동작 중...");
    }
}
```

### 3. 다른 클래스에서 사용 예시

```
public class Main {
    public static void main(String[] args) {
        // Singleton s = new Singleton(); // (X) 컴파일 에러 발생
        
        // 유일한 인스턴스 획득
        Singleton instance1 = Singleton.getInstance();
        Singleton instance2 = Singleton.getInstance();

        // 4. 동일성 검증
        if (instance1 == instance2) {
            System.out.println("두 객체는 메모리 주소가 동일한 '같은 인스턴스'입니다.");
        }
        
        instance1.doSomething();
    }
}
```
